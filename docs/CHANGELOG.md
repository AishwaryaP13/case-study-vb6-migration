### Schema decisions made

- **Dropped** `Invoices`, `InvoiceDetails`, `Warehouses` — 0 rows, no writing forms in VB6 original.
- **Added real FK constraints** on all implied relationships (none existed in the original SQLite schema).
- `Products.ProductID` kept as `TEXT` primary key (matches source data).
- `Users.Password` stored as bcrypt hash; seed data loaded with FK checks disabled during import.
- `StockLog.DocType`/`DocID` left without FK — polymorphic reference, no clean FK target possible.
